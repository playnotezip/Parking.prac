# UI/UX 디자인 가이드라인: 자동차 주차 연습 시뮬레이션

본 가이드라인은 제품 요구사항 정의서(PRD) 및 정보 구조서(IA)에 기반하여 작성되었습니다. 전체적인 비주얼 톤앤매너는 전문적이고 깔끔함(Mood)을 유지하면서도, 시뮬레이션 고유의 경쾌함을 살린 **재기발랄한(Playful)** 요소들을 인터랙션과 UI 포인트 컬러를 통해 녹여내어 초보 운전자의 심리적 장벽을 낮추는 것을 목표로 합니다.

---

## 1. 디자인 시스템 개요 (Design System Overview)

* **디자인 방향성**: 3D 그래픽과 복잡한 정보가 오가는 시뮬레이션의 특성을 고려하여, 핵심 대시보드와 UI 요소는 극도로 **깔끔하고 전문적인 레이아웃**으로 구성합니다. 대신 볼드한 타이포그래피, 라운드 처리가 강조된 컴포넌트 셰이프, 위트 있는 마이크로 인터랙션을 가미하여 **Playful 스타일**을 구현합니다.
* **그리드 및 레이아웃**: 모든 웹 페이지는 Tailwind CSS 표준의 12컬럼 그리드 시스템(`grid-cols-12`)을 기본 뼈대로 삼으며, 해상도에 따라 유연하게 컬럼 스팬(Column Span)을 변경하는 반응형 구조를 채택합니다.
* **접근성 및 시인성**: 3D 화면과 오버레이 UI 간의 간섭을 줄이기 위해 투명도(`backdrop-blur`)를 적절히 활용하고, 야간 주차 및 지하 주차장 맵의 어두운 톤에서도 가독성을 잃지 않도록 보색 대비를 철저히 검증합니다.

## 2. Tailwind CSS 컬러 팔레트 (Color Palette)

주어진 Primary 500 컬러(`#00ffc7`)를 기준으로 색상환에서 정삼각형을 이루는 삼색 배색(Triadic) 체계를 구축했습니다. 메인 컬러의 세련된 청록빛을 전문적인 차콜 톤이 받쳐주고, 개성 있는 퍼플과 경고성 옐로우가 포인트 역할을 수행합니다.

| 분류 | Tailwind 명칭 | 색상 코드 (Hex) | 디자인 결정 이유 (Rationale) |
| --- | --- | --- | --- |
| **Primary** | `bg-primary-500` | `#00ffc7` | 청량하고 미래지향적인 네온 민트 컬러입니다. 주차 성공 상태, 핵심 실행 버튼(CTA)에 적용하여 플레이풀한 아이덴티티를 각인시킵니다. |
| **Secondary** | `bg-secondary-500` | `#ffc700` | 따뜻하면서도 눈에 띄는 생생한 옐로우 컬러입니다. 미러 위젯의 가이드라인, 주차 점수 강조, 차량의 조작 가이드 텍스트에 적용합니다. |
| **Accent** | `bg-accent-500` | `#c700ff` | 강렬하고 신비로운 바이올렛 퍼플 컬러입니다. 현재 기어 상태(`P-R-N-D`) 표시 위젯, 마우스 휠 인터랙션 하이라이트에 사용합니다. |
| **Neutral Dark** | `bg-neutral-900` | `#111215` | 깊이 있는 블랙 그레이로, 전체 3D 화면을 감싸는 배경과 상단 바에 적용되어 "전문적이고 깔끔한" 인상을 부여합니다. |
| **Neutral Light** | `bg-neutral-100` | `#f4f6f9` | 텍스트 및 카드 컴포넌트의 밝은 배경으로 사용되어 다크 톤과 명확한 대비를 이룹니다. |
| **System Error** | `bg-danger-500` | `#ff4a4a` | 벽면 또는 타 차량 충돌 시 UI 테두리가 깜빡이는 경고 컴포넌트에 한정적으로 사용합니다. |

## 3. 중단점 설정 (Breakpoints)

제시된 고정 브레이크포인트를 반영하여 미디어 쿼리 및 Tailwind 단계를 매핑합니다. 데스크톱 위주의 시뮬레이터 런타임 특성을 반영하여 레이아웃 변형을 설계합니다.

```text
mobile:  320px  // 스마트폰 세로 모드 최저 기준
tablet:  768px  // 태블릿 디바이스 기준
desktop: 1024px // 일반 노트북 및 모니터 해상도 (표준 레이아웃)
wide:    1440px // 고해상도 데스크톱 모니터 최적화

```

## 4. 공통 레이아웃 컴포넌트 (Layout Components)

### 4.1. 글로벌 상단 바 (Topbar Navigation)

* **적용 라우트**: 루트 경로(`/`), `/simulation`, `/guide`, `/mypage`, `/auth/*` 전체 공통 적용
* **핵심 컴포넌트**: 서비스 로고, 메인 메뉴 그룹(시뮬레이션 연습, 주차 가이드), 사용자 상태 영역(마이페이지 아이콘 또는 로그인 버튼)
* **반응형 동작**:
* `desktop` 이상: 12컬럼 중앙 정렬 구조로 모든 내비게이션 메뉴가 가로로 나열됩니다.
* `tablet` 이하: 메뉴 그룹이 숨겨지고 우측에 햄버거 메뉴 버튼(`bg-neutral-900` 위의 `text-white` 삼선 아이콘)이 등장하며, 클릭 시 우측 슬라이드 드로어가 활성화됩니다.

## 5. 페이지별 UI/UX 구현안 (Page Implementations)

### 5.1. 메인 홈 페이지 (`/`)

* **핵심 목적**: 비로그인 사용자를 포함한 모든 진입 유저에게 시뮬레이션의 목적을 명확히 전달하고, 즉시 플레이 환경으로 진입할 수 있도록 유도합니다.
* **레이아웃 구조**:
* `desktop`: 12컬럼 중 좌측 6컬럼은 텍스트 및 CTA 버튼, 우측 6컬럼은 메인 비주얼 일러스트가 배치되는 대칭형 구조입니다.


* **핵심 UI 컴포넌트 및 가시화**:
* **메인 카피**: "면허는 있지만 주차는 무서운 당신을 위해, 안전한 가상 공간에서 완벽하게 마스터하세요." (`text-white`, `font-bold`)
* **바로가기 버튼**: `bg-primary-500` 컬러가 적용된 구부러진 라운드 형태의 큰 버튼으로, 마우스 오버 시 `scale-105`로 튕기듯 커집니다.
* **시각 자료(Hero Image)**: 실제 가상 주차 공간의 분위기를 보여주기 위해 정갈한 도로 및 차량 구도의 스탁 이미지를 우측 패널 배경으로 배치합니다.



### 5.2. 시뮬레이션 환경 설정 페이지 (`/simulation`)

* **핵심 목적**: 사용자가 주차 연습을 시작하기 전, 원하는 차종 규격과 연습 장소(맵)를 플레이풀한 카드 인터페이스를 통해 직관적으로 선택하도록 돕습니다.
* **레이아웃 구조**:
* 상단 3컬럼 단위의 차종 선택 카드 영역(소형, 세단, SUV) + 하단 4컬럼 단위의 주차 맵 선택 영역(아파트 지하, 대형마트, 골목길).


* **핵심 UI 컴포넌트 및 가시화**:
* **차종 선택 카드**: 선택 시 테두리에 `border-4 border-accent-500` 스태킹 선이 생기며 카드가 위로 통통 튀어 오르는 효과를 줍니다. 내부에는 전장/전폭 수치가 깔끔한 라벨 그리드로 표시됩니다.
* **시각 자료(Vehicle Concept)**: 차종 선택의 직관성을 위해 세단 및 도시적인 차량 이미지를 카드 컴포넌트 내부에 삽입합니다.



### 5.3. 시뮬레이터 실행 룸 페이지 (`/simulation/[roomId]`)

* **핵심 목적**: WebGL 3D 그래픽을 통해 실제 운전석 또는 탑뷰 시점을 제공하고, 키보드 조작과 실시간 피드백을 인터랙티브하게 매핑합니다.
* **레이아웃 구조**: Full-screen 3D 캔버스를 기반으로 하며, 주변부에 HUD(Head-Up Display) 스타일의 위젯들이 12컬럼의 그리드 외곽 좌우측 영역에 오버레이(`absolute`)됩니다.
* **핵심 UI 컴포넌트**:
* **가상 사이드미러 위젯 (좌/우 상단)**: 후방 시야를 실시간 렌더링하는 사각형 위젯입니다. 테두리는 깔끔한 차콜 톤이지만 후방 장애물이 1m 이내로 접근 시 `bg-secondary-500` 색상으로 깜빡입니다.
* **기어 변환 레버 (우측 하단)**: 마우스 휠 동작에 반응하여 `P - R - N - D` 알파벳이 위아래로 부드럽게 스크롤링되며, 현재 기어는 `text-accent-500`과 함께 볼드 처리됩니다.
* **실시간 조작 가이드 (하단 중앙)**: `W`, `A`, `S`, `D` 키가 화면에 키캡 모양으로 시각화되어 있으며, 유저가 키를 누를 때마다 해당 키캡 UI가 네온 청록색(`bg-primary-500`)으로 밝게 빛납니다.

## 6. 인터랙션 패턴 (Interaction Patterns)

* **기어 휠 스내핑 (Mouse Wheel Snapping)**: 마우스 휠을 한 칸 굴릴 때마다 기어 게이지가 미끄러지듯 이동하다가 정확한 단수 위치에 툭 걸리는(Snapping) 물리적인 모션을 부여하여 손맛을 극대화합니다.
* **바운시 버튼 효과 (Bouncy Hover & Click)**: 모든 클릭 가능한 플레이풀 버튼 요소들은 마우스 오버 시 `translate-y-[-4px]`, 클릭 시 `scale-95`로 빠르게 수축했다가 복원되는 스프링 애니메이션 효과를 일관되게 적용하여 조작의 피로도를 낮추고 유쾌한 감각을 전달합니다.
* **충돌 경고 피드백 (Crash Flash)**: 3D 가상 공간 내에서 장애물과 조그마한 충돌이라도 발생할 경우, 화면 전체 가장자리에 `inset-0 border-8 border-danger-500` 레이어가 0.3초간 빠르게 두 번 점멸하여 유저가 실수를 직관적으로 인지하되 깔끔한 경고 시스템으로 기능하도록 설계합니다.
