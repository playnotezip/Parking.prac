# 유스케이스 명세서 (Use Case Specification)

본 문서는 앞서 정의된 제품 요구사항 정의서(PRD) 및 정보 구조서(IA)를 바탕으로, **웹 기반 자동차 주차 연습 시뮬레이션** 시스템의 핵심 기능에 대한 유스케이스를 상세히 정의합니다.

---

## 1. 액터 정의 (Actor Definitions & Comprehensive Actor Definitions)

| 액터명 | 분류 | 설명 |
| --- | --- | --- |
| **비로그인 사용자 (Guest User)** | 주 액터 (Primary Actor) | 회원가입이나 로그인 없이 웹사이트에 접속하여 주차 시뮬레이션을 체험하고 주차 공식 가이드를 조회하는 일반 사용자 |
| **로그인 사용자 (Registered User)** | 주 액터 (Primary Actor) | 본인 계정으로 인증을 완료한 유저로, 비로그인 사용자의 모든 권한을 포함하며 추가로 주차 연습 이력 저장 및 마이페이지 대시보드 조회 권한을 가짐 |
| **OpenAI API** | 부 액터 (Supporting Actor) | 사용자의 주차 시뮬레이션 로그 데이터를 전달받아 자연어 기반의 맞춤형 한국어 교정 피드백을 생성하는 외부 AI 시스템 |
| **데이터베이스 (Database)** | 부 액터 (Supporting Actor) | 사용자 정보, 차종/맵 설정 값, 주차 판정 점수 및 OpenAI가 생성한 피드백 이력을 영속적으로 저장하고 관리하는 내부 시스템 |

---

## 2. 유스케이스 요약 및 시나리오 (Detailed Use Case Scenarios)

본 문서에서 다룰 핵심 유스케이스는 'UC-01: 주차 시뮬레이션 및 AI 피드백 확인'입니다. 이 유스케이스는 사용자가 차량과 맵을 선택해 주차를 수행하고 AI 피드백을 받는 전체 핵심 흐름을 관장합니다.

---

## 3. 사전 조건 및 사후 조건 (Preconditions and Postconditions)

* **사전 조건 (Preconditions)**:
* 사용자는 웹 브라우저를 통해 시스템의 주차 설정 페이지(`/simulation`)에 정상적으로 접속한 상태여야 합니다.
* OpenAI API 및 내부 데이터베이스 시스템이 정상 작동 상태여야 합니다.


* **사후 조건 (Postconditions)**:
* **성공**: 주차 결과 데이터가 계산되어 화면에 출력되고, OpenAI가 생성한 한국어 피드백이 표시됩니다. 로그인 유저의 경우 이 데이터가 데이터베이스에 영속적으로 저장됩니다.
* **실패**: 시스템 오류 발생 시 유저에게 에러 메시지를 노출하고 기존 데이터 상태를 유지합니다(Rollback).



---

## 4. 기본 흐름 (Main Steps and Flow of Events)

1. **설정 선택**: 사용자는 화면에서 연습하고자 하는 차종(소형, 세단, SUV)과 주차 맵(아파트, 마트 등)을 선택한 뒤 [시뮬레이션 시작] 버튼을 누릅니다.
2. **시뮬레이터 로드**: 시스템은 Next.js 환경에서 WebGL 3D 캔버스 화면을 초기화하고, 선택된 차량 및 맵 에셋을 로드합니다.
3. **차량 조작**: 사용자는 키보드(`W`, `A`, `S`, `D`)로 가속/조향/브레이크를 조작하고, 마우스 휠로 기어(`P-R-N-D`)를 변경하며 주차를 수행합니다. 필요에 따라 `V` 키로 1인칭/3인칭 시점을 교체합니다.
4. **주차 완료 요청**: 사용자가 차량을 주차 공간 내에 정렬한 후 기어를 `P(주차)`로 변경하면 시뮬레이션이 종료됩니다.
5. **데이터 분석**: 시스템은 주차 소요 시간, 충돌 횟수, 최종 차량 정렬 각도 및 선 침범 여부 데이터를 수집합니다.
6. **피드백 요청**: 시스템은 수집된 정량적 로그 데이터를 가공하여 부 액터인 OpenAI API에 전달하며 한국어 분석 리포트 생성을 요청합니다.
7. **결과 화면 표출**: 시스템은 계산된 주차 점수와 OpenAI로부터 수신한 한국어 코칭 멘트를 결과 모달 팝업 창에 출력합니다.
8. **저장 처리 (로그인 유저 상태)**: 유저가 로그인 상태인 경우, 시스템은 점수 및 피드백 텍스트를 데이터베이스에 자동으로 저장합니다.

---

## 5. 대체 흐름 및 예외 처리 (Alternative Flows, Edge Cases, and Exception Handling)

### 5.1. 대체 흐름 (Alternative Flows)

* **Alt 1. 비로그인 유저의 결과 저장 시도**:
* 기본 흐름 7단계 이후, 유저가 비로그인 상태이면 시스템은 결과 화면 하단에 [기록 저장하기] 버튼을 노출합니다.
* 유저가 이를 클릭하면 "기록을 저장하려면 로그인이 필요합니다. 로그인 페이지로 이동하시겠습니까?" 팝업을 띄우고 회원가입/로그인 동선으로 유도합니다.


* **Alt 2. 제한 시간 초과 (Edge Case)**:
* 주차 시작 후 지정된 제한 시간(예: 5분)이 초과되면 기어 변경 여부와 상관없이 시뮬레이션이 강제 종료되고 '시간 초과(0점)' 판정과 함께 기본 흐름 5단계로 이동합니다.



### 5.2. 예외 처리 절차 (Exception Handling Procedures)

* **Exc 1. OpenAI API 연동 실패 또는 타임아웃**:
* 기본 흐름 6단계에서 네트워크 장애나 API 제한으로 인해 피드백 생성에 실패하는 경우입니다.
* 시스템은 OpenAI 피드백 영역에 "AI 피드백 로딩에 실패했습니다. 기본 점수 리포트만 표출합니다."라는 에러 안내 문구를 한국어로 노출하고, 정량적 점수(시간, 충돌 횟수)만 모달창에 정상 표출하여 유저 경험 단절을 막습니다.


* **Exc 2. WebGL 렌더링 컨텍스트 상실**:
* 사용자의 브라우저 사양 문제나 그래픽 드라이버 이슈로 3D 화면이 멈추거나 깨지는 경우입니다.
* 시스템은 "그래픽 카드가 3D 환경을 지원하지 않거나 오류가 발생했습니다. 브라우저를 새로고침하거나 크롬(Chrome) 브라우저 이용을 권장합니다."라는 경고 메시지를 팝업으로 안내합니다.

## 6. 비즈니스 규칙 및 제약 조건 (Business Rules and Constraints)

* **점수 산정 규칙**: 주차 점수는 100점 만점에서 시작하여 감점 방식으로 계산합니다.
* 벽면 및 타 차량 충돌 1회당: -20점
* 주차선 침범 유지가 지속될 경우 1초당: -2점
* 제한 시간 이내 완수 시 남은 시간에 비례한 보너스 점수 부여 (최대 +10점)


* **종료 조건의 엄격성**: 기어가 `P` 상태이고, 차량 속도가 `0`인 상태가 1.5초 이상 유지되어야 정상적인 주차 완료 상태로 인정됩니다. 주행 중 실수로 `P` 기어를 스크롤 하더라도 즉시 종료되지 않도록 방지합니다.

---

## 7. 사용자 인터페이스 고려사항 (User Interface Considerations)

* **반응형 조작 안내 UI**: 화면 레이아웃 우측 상단에 현재 활성화된 키보드 키(`W, A, S, D`)가 하이라이트되는 시각적 위젯을 배치하여 조작 직관성을 높입니다.
* **모바일/태블릿 접근성 제약**: 본 시뮬레이션은 데스크톱 키보드 조작과 마우스 휠 제어를 핵심 인터랙션으로 삼고 있으므로, 모바일 기기로 접근 시 화면 중앙에 "본 시뮬레이션은 키보드와 마우스가 장착된 데스크톱 환경에 최적화되어 있습니다"라는 모달 안내 팝업을 상시 배치합니다.
* **웹 접근성(A11y)**: 결과 모달 팝업이 등장할 때 포커스(Focus)가 모달 내부로 강제 이동하게 하여 스크린 리더 사용자가 점수와 피드백 텍스트를 순차적으로 읽을 수 있도록 `aria-modal="true"` 속성을 지정합니다.

## 8. 데이터 요구사항 및 데이터 흐름 (Data Requirements and Data Flow)

### 8.1. 시뮬레이션 종료 시 발생 데이터 형식

시스템 내부적으로 수집되어 OpenAI API 및 DB로 전송되는 데이터 스키마 예시 구조입니다.

```json
{
  "userId": "user_12345_optional",
  "carType": "sedan",
  "mapId": "apartment_underground",
  "metrics": {
    "elapsedTimeSeconds": 112,
    "collisionCount": 1,
    "lineViolationDurationSeconds": 4.5,
    "finalAngleOffsetDegree": 2.3
  }
}

```

### 8.2. 데이터 흐름 (Data Flow)

1. **[클라이언트 WebGL]** 사용자가 주차를 완료하여 발생한 원시 주행 데이터(Metrics) 수집
2. **[Next.js 서버 라우트]** 원시 데이터를 가공하여 OpenAI 프롬프트 템플릿(한국어 설정) 결합
3. **[OpenAI API]** 프롬프트를 분석하여 맞춤형 주차 피드백 텍스트 응답 반환
4. **[Next.js 서버 라우트]** 반환된 텍스트와 원시 데이터를 병합하여 클라이언트에 송신 및 (로그인 시) 내부 데이터베이스에 적재

## 9. 보안 및 개인정보 보호 고려사항 (Security and Privacy Considerations)

* **선택적 로그인 보안**: 비로그인 사용자의 데이터는 서버 세션이나 쿠키에 임시 식별자(UUID) 형태로 유지되며, 회원가입 전환 시 해당 데이터가 유저 계정으로 안전하게 귀속되도록 매핑 로직을 보호합니다.
* **OpenAI API 프롬프트 인젝션 방지**: 클라이언트가 전송하는 데이터를 백엔드 서버 라우트에서 철저히 검증(Validation)한 후 OpenAI API에 바인딩합니다. 클라이언트가 임의로 프롬프트를 수정하여 전달할 수 없도록 차단합니다.
* **개인정보 최소화**: 본 서비스는 운전 연습 기능을 골자로 하므로 회원가입 시 주민등록번호나 운전면허번호 등 불필요한 민감 고유식별정보를 절대 수집하지 않으며, 이메일과 비밀번호만을 입력받아 운영합니다.
